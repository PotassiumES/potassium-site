🌸 PotassiumES 🌸

Introduction

This is the story of PotassiumES, a free and open tool for making amazing sites on the wider web. If you're not sure about the wider web (and few people are yet) then click that link for a two minute overview.

For the twitchy: This is a quick read but if you'd prefer to hop right into making the wider web then click over to the samples repo, core repo, or the docs.

Designers, Artists, Writers, Coders, and the PMs who 💖 them.

XR (augmented and virtual reality) is coming to browsers and that means that many new sorts of people will be involved in making the wider web. PotassiumES welcomes these new teams with wide(r) open arms.

In the sections below you'll find brief introductions to each role and how PotassiumES gives us all super-powers:

  • UX and Visual Design
  • Spatial Graphics and Audio
  • Narrative and Information
  • Product and Project Management
  • Software Engineering and Ops

User Experience and Visual Design

What designers do

User experience designers...

  • Understand and represent the user
  • Nail down important scenarios and workflows
  • Define the user's journey from goal to success

Visual designers...

  • Understand the user and how they process information
  • Define a visual language with space, color, shape, and fonts
  • Design and communicate visual elements with the team

Challenges on the wider web

If we thought that adding responsive layouts for mobile and PC screens was extra work, what if we add two radically different display types: handheld portal displays and immersive headsets?!

Now that the users can browse on a mobile phone, a laptop, a tablet that looks onto an augmented world, and a headset with a floating browser, how do we create user flows that work across all of these platforms?

Design systems are all the rage but not one of them address UI elements that overlay a portal display or spatial controls that respond to tracked controllers, hand gestures, and voice commands. How do we even spec out sizes and animations for 3D elements?

What PotassiumES provides

Component library: Browsers provide a lot of support for flat page UI elements. Buttons, checkboxes, dropdowns, and more come "for free". Unfortunately, none of that is true for overlay and spatial controls so PotassiumES provides a set of pre-made UI components that provide overlay and spatial controls "for free."

You don't have to start from scratch when designing 3D!

PotassiumES Style System (KSS): In the last few years CSS has added flexbox and grid layouts, making all of our lives easier. We want to ride that wave so KSS provides the same syntax and control as CSS but for overlay and spatial controls.

You can use KSS to style 2D and 3D just like CSS!

Input abstraction: No longer can we respond to just keyboards, mice, and touchscreens. There is a wave of new input hardware and software coming and the only way to handle the constant change is to take in the tsunami of input and connect it to higher level actions. PotassiumES uses the excellent Mozilla Action-Input library to solve this problem.

When new input types arrive, you'll be ready!

Sample apps: Hand waving is fine and all, but nothing is as real as poking and prodding working examples to get a feel for a new tool. PotassiumES's sample applications can be set up on your Mac or PC in about 15 minutes and they provide links to all of the documentation that you'll need to get started.

Spatial Graphics and Audio

What 3D artists and sound designers do

3D artists...

  • Sculpt and model hard and organic shapes
  • Create materials using images and code
  • Rig and animate characters and environments

Sound designers...

  • Generate and record sounds for events and environments
  • Orchestrate effects and tracks that react to user actions
  • Mix for a wide variety of playback hardware

Challenges on the wider web

The flat web is quiet and people get mad over autoplay. Headsets offer a new venue for audio but how do we know when to use sound and what type to use?

Game consoles and fast-GPU PCs can be complex to support and the backlit screens in mobile phones and laptops are relatively simple compared to additive light field displays in augmented reality glasses and the optics in virtual reality headsets. Will we need to use separate workflows for each type of display and GPU?

What PotassiumES Provides

Standard formats: PotassiumES includes a growing number of high quality importers that read the most common file formats for 3D and audio and then render them efficiently and beautifully. On less capable hardware it tries to degrade gracefully.

You don't have to learn all new editors and file formats!

Fast iteration: Unlike native XR application tools, the web doesn't make you wait around for hours while you build a binary, put it on a device, and then fire it up. The PotassiumES build system is so fast that it's usually done before you have a chance to reload the page!

No more going for coffee while the app builds!

(unless you want to)

Unified UIs and APIs: Designing and creating separate experiences for different display modes, input types, and locations in the world is too hard if we're using disconnected frameworks and tools. PotassiumES brings together the essential concepts of the wider web and gives you a single consistent framework to share your work.

Narrative and Information

Storytellers...

  • Create worlds, histories, and characters
  • Check for consistency across the entire experience
  • Bridge emotional connections to users

UI writers...

  • Write copy for UI elements
  • Prevent badd typoos and error grammars
  • Create and maintain the "voice" of the site

Challenges on the wider web

Text is often hard to read in immersive headsets because the display technology isn't great or because the user has literally walked away. Writing for spatial experiences can be more like writing for architectural signage than writing for the flat web.

Emergant narratives that work in immersive environments may not work on the flat web where linear stories still reign. How do we take advantage of the emotional thrill of immersive stories when the user can switch to flat display mode at any time?

The wider web is thoroughly global with most of the growth coming from regions with different languages, cultures, time zones, currencies, alphabets, and beliefs. The political divide and economic gap in the US is a drop in the bucket (a confusing phrase to non-English speakers). How do we write for the next billion web users?

What PotassiumES Provides

Localization and Translation: To reach the next billion web users it takes a team of people to translate between languages and cultures. PotassiumES provides a smooth path through translation services, copy editing, and eventually to publication.

Writers are welcome on the wider web.

Voice commands: Words are the lingua franca of the flat web but the wider web listens to what we say. PotassiumES provides you with tools for the site to listen for spoken phrases and then map them to higher level user actions.

Product and Project Management

Product managers...

  • Gather product and customer requirements
  • Ensure that a product is continuously successful
  • Hold a long term view to coordinate overall direction

Project managers...

  • Manage risk and unexpected challenges
  • Coordinate projects' schedules and resources
  • Align designers, artists, writers, and coders

Challenges on the wider web

When the web adds big new platforms it usually causes a wave of tools that ask you to replace what already exists. The web is too big to throw away and start over no matter how interesting or valuable XR could be.

The next billion web users will come from new markets where you don't speak the language and don't understand the culture. How will these new users learn to use technologies like XR headsets at the same time that they learn to use the flat web?

We've all spent years (and sometimes tears) to foster our people and to make certain that they're at the top of their field. To throw away all of that experience and to risk losing people because of too much change can be a project killer.

What PotassiumES Provides

Don't boil the ocean: PotassiumES is built using the most common front-end toolchain. An engineer can probably add PotassiumES to your existing build in less than a day and then add features on your schedule.

You can use what you have while you test the waters!

Open up global markets: PotassiumES includes highly effective localization and translation APIs that work well with the tools used by third party localization services. It also takes advantage of off-the-shelf, web-scale infrastructure such as load balancers, content delivery networks, and the cloud based continuous integration that you'll need once the entire world is your userbase.

Your ops team already knows how to deploy the web.

Retraining is not an option: PotassiumES is designed to be familiar to your existing team members and welcoming to people from other industries like 3D artists, sound designers, and immersive experience designers.

Empower your current team and attract new people.

Software Engineering and Ops

Product engineers...

  • Create new features
  • Maintain the health of the stack
  • Communicate technical possibilities and constraints

Operations engineers...

  • Create stable deployments
  • Maintain the health of the service
  • Communicate operational possibilities and constraints

Challenges on the wider web

Wider web applications often integrate with uniquely complex third party services for environment recognition in augmented reality and with avatar services for virtual reality.

For multiuser applications, new infrastructure is required to calculate and replicate complex simulations underlying applications in which even mild network lag and latency is unacceptable.

Designers and PMs are learning how to communicate wider web designs and requirements so it can be hard to even understand what needs to be done!

What PotassiumES Provides

The core technologies of PotassiumES are off-the-shelf and will easily integrate with your current front-end builds.

You already know how to build PotassiumES.

The component library that ships with PotassiumES is loaded with well commented examples for many common engineering tasks.

You won't need to start from scratch.

Every single aspect of PotassiumES is developed in public under the most permissive open source license and with the goal of fostering a healthy community of contributors.

You can always "view source".
